1. API GATEWAY
--------------------------------------------------------------------------------

[Câu] API versioning (v1/v2/test)
  Đáp án: Stages + Stage Variables
  Tip: Authorizer = auth; Resource Policy = access control — KHÔNG dùng versioning
  Tip: Mỗi stage có URL riêng; stage var trỏ Lambda alias

[Câu] prod/test/dev → Lambda khác nhau
  Đáp án: Stage Variables + Lambda Aliases
  Tip: ARN: ...function:MyFn:${stageVariables.lambdaAlias}

[Câu] Method-level execution logging
  Đáp án: IAM role (trust apigateway.amazonaws.com) + STS enabled ở region
  Tip: Không nhầm execution logging vs access logging

[Câu] Custom headers + DynamoDB auth
  Đáp án: Lambda Authorizer (REQUEST type)
  Tip: Model = validate JSON; Cognito ≠ lookup DynamoDB custom

[Câu] HTTP error codes
  Đáp án: 503 = no target; 502 = bad response; 504 = timeout
  Tip: "Không biết gửi đi đâu" → 503

[Câu] Access control API Gateway — cơ chế KHÔNG hỗ trợ (NOT supported)
  Yêu cầu đề: Lambda behind API GW; đề hỏi cơ chế kiểm soát truy cập nào API GW không hỗ trợ
  Đáp án: STS — không phải authorizer/access mechanism của API Gateway
  Tip: Hỗ trợ: IAM SigV4, Cognito User Pools, Lambda Authorizer, API Keys, Resource policy
  Lỗi hay nhầm: Chọn IAM SigV4 — đây LÀ cơ chế hỗ trợ; STS chỉ service cấp temporary creds (dùng bởi IAM/role)

--------------------------------------------------------------------------------

[Câu] Node.js + dependencies (pg/RDS)
  Đáp án: Zip chung code + node_modules trong 1 file
  Tip: Lambda KHÔNG tự npm install; không upload 2 zip riêng

[Câu] Cross-account → DynamoDB (Account B)
  Yêu cầu đề: Lambda Account A cần đọc/ghi DynamoDB Account B
  Yêu cầu đề: Đề hỏi cấu hình IAM thế nào; DynamoDB không có resource policy
  Yêu cầu đề: Bẫy — sửa trust policy execution role A, hoặc policy trực tiếp lên table
  Đáp án: AssumeRole; trust policy trên role Account B
  Tip: Trust policy thuộc role ĐÍCH, không phải execution role A

[Câu] DLQ khi nào nhận message?
  Đáp án: Async invocation + fail HẾT mọi retry
  Tip: Sync (API GW) không dùng DLQ async; fail 1 lần rồi OK → không vào DLQ

[Câu] Bundle dependencies
  Đáp án: 1 deployment package hoặc Lambda Layers

3. S3
--------------------------------------------------------------------------------

[Câu] Private bucket, truy cập tạm thời
  Đáp án: Pre-signed URL
  Tip: Bucket/IAM policy = quyền lâu dài

[Câu] Bắt buộc HTTPS (in transit)
  Đáp án: Bucket policy Deny + aws:SecureTransport: false
  Tip: KMS policy không enforce HTTPS cho GetObject

[Câu] Static website 403
  Đáp án: Bucket policy public s3:GetObject
  Tip: 403 = permission; CORS = cross-origin browser

[Câu] S3 mới, CloudFront cũ
  Đáp án: CloudFront invalidation
  Tip: CI/CD: upload S3 → invalidate CF

[Câu] Overwrite + đọc ngay
  Đáp án: Luôn trả version mới nhất (strong consistency từ 2020)
  Tip: Từ "might/propagate" = behavior cũ (SAI)

[Câu] Event notification thiếu
  Đáp án: Concurrent PUT cùng key, non-versioned → có thể chỉ 1 event
  Tip: Versioning GIÚP, không phải nguyên nhân thiếu event

[Câu] Cross-account NOT feasible
  Đáp án: Khác partition (aws vs aws-cn)
  Tip: Cùng partition → bucket policy / AssumeRole OK

[Câu] Multi-account upload, bucket owner
  Đáp án: S3 Object Ownership (Bucket owner enforced)
  Tip: CORS/Access Analyzer không đổi ownership

[Câu] Subset CSV (1 tháng, 3 cột)
  Yêu cầu đề: CSV lớn trên S3; lấy 1 tháng + 3/10 cột; minimize compute & network cost
  Đáp án: S3 Select
  Tip: Filter/projection trên S3 — không tải cả file; Analytics/Inventory/Access Logs không query nội dung CSV
  Lỗi hay nhầm: S3 Analytics (storage class); Inventory (metadata); Access Logs (audit)

[Câu] S3 access control (chọn 2)
  Đáp án: Bucket policy + IAM; Pre-signed URL + ACLs
  Tip: Permissions boundaries ≠ S3 access

[Câu] Transfer Acceleration
  Đáp án: Tăng tốc upload/download từ xa qua edge
  Tip: Không replicate; khác CloudFront cache

[Câu] On-prem upload file nén ~2 GB lên S3 backup hàng ngày — nhanh nhất
  Yêu cầu đề: Data center on-premises; file ~2 GB/ngày; hỏi cách upload S3 nhanh nhất
  Yêu cầu đề: File lớn + đường truyền từ xa → cần multipart VÀ tối ưu path
  Đáp án: Multipart upload + S3 Transfer Acceleration
  Tip: Multipart alone thiếu khi đề nhấn on-prem/xa; single PUT chậm; FTP→EC2→S3 thêm hop
  Lỗi hay nhầm: Chỉ multipart (không TA); single operation; relay qua EC2

[Câu] SSE-S3 vs SSE-KMS vs SSE-C
  Yêu cầu đề: Chọn loại mã hóa at rest; ai quản lý key
  Đáp án: SSE-S3 = S3 AES-256; SSE-KMS = KMS CMK; SSE-C = client gửi key mỗi request
  Tip: SSE-S3 đơn giản không KMS cost; SSE-KMS có audit CloudTrail
  Tip: SSE-C S3 không lưu key; bắt buộc HTTPS; mất key = mất data

[Câu] SSE-KMS — khóa KMS & quyền truy cập
  Yêu cầu đề: Access Denied object SSE-KMS; cross-account cần KMS key policy
  Đáp án: IAM cần S3 + kms:Decrypt/kms:GenerateDataKey; key policy allow principal
  Tip: aws/s3 default CMK vs customer managed CMK

[Câu] SSE-C — customer-provided keys
  Yêu cầu đề: Tự quản lý key ngoài AWS; header x-amz-server-side-encryption-customer-*
  Đáp án: Client gửi key mỗi PUT/GET; S3 không persist key
  Tip: Copy object SSE-C phải cung cấp key nguồn/đích

[Câu] Default encryption & bắt buộc mã hóa
  Yêu cầu đề: Mọi upload phải SSE; không cho unencrypted object
  Đáp án: Default encryption bucket + bucket policy Deny nếu thiếu SSE header
  Tip: HTTPS (SecureTransport) ≠ SSE at rest — 2 policy khác nhau

[Câu] Storage classes — tổng quan
  Yêu cầu đề: Chọn class theo tần suất truy cập, retrieval time, chi phí
  Đáp án: Standard → IA → Glacier Instant/Flexible → Deep Archive
  Tip: Standard-IA/One Zone-IA min 30 ngày + retrieval fee
  Tip: Intelligent-Tiering auto tier, phí monitoring, không retrieval fee

[Câu] Chọn storage class theo use case
  Yêu cầu đề: Archive năm/chờ giờ/ms; pattern không rõ; chấp nhận 1 AZ
  Đáp án: Deep Archive rẻ nhất lâu dài; GIR ms retrieval; Intelligent-Tiering pattern không rõ
  Tip: One Zone-IA rẻ hơn Standard-IA nhưng chỉ 1 AZ durability

[Câu] Lifecycle — chuyển storage class
  Yêu cầu đề: Tự chuyển IA/Glacier sau N ngày; xóa sau retention
  Đáp án: S3 Lifecycle rules — Transition + Expiration
  Tip: Glacier restore tạo bản copy tạm; không transition ngược trực tiếp

4. CLOUDFRONT
--------------------------------------------------------------------------------

[Câu] Route S3 theo country (cache miss)
  Đáp án: Lambda@Edge + origin request
  Tip: CloudFront Function chỉ viewer req/resp; origin request = cache miss

[Câu] Origin Groups failover — video streaming HA (chọn 2)
  Yêu cầu đề: CloudFront Origin Group primary + secondary; high availability / origin failover
  Yêu cầu đề: Chọn 2 câu ĐÚNG về hành vi failover — không nhầm cache hit hay “all primary”
  Đáp án (2): (1) Mỗi request mới vẫn thử primary trước — kể cả request trước đã failover sang secondary
  Đáp án (2): Failover sang secondary chỉ khi HTTP method = GET, HEAD, OPTIONS (không POST/PUT/DELETE…)
  Tip: Cache hit → trả từ edge cache, không gọi origin (SAI nếu nói route primary khi cache hit)
  Tip: Origin group = 1 primary + 1 secondary — không phải “tất cả đều primary”
  Tip: Chỉ cần ≥2 origins (1 group: primary + secondary) — không bắt buộc 3 origins
  Lỗi hay nhầm: All origins primary; cache hit route primary origin

[Câu] Auth offload (CF→ALB→EC2)
  Đáp án: Cognito User Pools trên ALB
  Tip: CloudFront không có User Pool auth built-in

[Câu] EC2 đa ngôn ngữ ?lang= — CloudFront luôn 1 ngôn ngữ
  Yêu cầu đề: Origin EC2 đúng theo query string lang; CloudFront cache sai — mọi lang ra cùng nội dung
  Yêu cầu đề: Cùng path/file name, khác query string → fix cache behavior
  Đáp án: Cache policy — Query string forwarding and caching; whitelist `lang`
  Tip: Query string vào cache key; không nhầm header whitelist; giảm TTL không fix gốc
  Lỗi hay nhầm: Cache theo headers; Cache None; chỉ giảm TTL

[Câu] Static S3 + dynamic ALB — latency thấp global
  Yêu cầu đề: Static từ S3, dynamic từ ALB; user global; minimize latency
  Yêu cầu đề: Một CDN phục vụ cả hai loại nội dung qua edge
  Đáp án: CloudFront multiple origins — behavior/path route S3 vs ALB
  Tip: CF cache static ở edge; dynamic forward ALB; Origin Groups = failover không gộp 2 origin thành 1 request
  Lỗi hay nhầm: Global Accelerator (TCP/UDP, không cache static web); Lambda@Edge route thủ công không cần khi multi-origin đủ

5. ECS / FARGATE / ECR
--------------------------------------------------------------------------------

[Câu] Serverless + 2 container — sidecar pattern
  Yêu cầu đề: Serverless + 2 container chia sẻ network; sidecar = container phụ cùng task
  Đáp án: 1 task definition + Fargate — 2 container definitions trong cùng task
  Tip: Sidecar gọi localhost; deploy/scale chung task; không 2 task definition riêng

[Câu] X-Ray daemon sidecar trên Fargate
  Yêu cầu đề: Fargate cần X-Ray traces; chọn 2: deploy daemon + IAM
  Đáp án: Sidecar container X-Ray daemon + Task IAM role (xray:PutTraceSegments…)
  Tip: Daemon Set chỉ EC2 launch; Fargate = sidecar; AWS_XRAY_DAEMON_ADDRESS
  Lỗi hay nhầm: Daemon Set trên Fargate

[Câu] Task Execution Role vs Task IAM Role
  Yêu cầu đề: Pull ECR / logs fail vs app gọi S3/DDB fail — role nào?
  Đáp án: Execution Role = pull ECR + awslogs; Task Role = app AWS API
  Tip: EC2 launch Task Role cần ECS_ENABLE_TASK_IAM_ROLE=true

[Câu] Task IAM role không hoạt động (EC2 launch type)
  Đáp án: ECS_ENABLE_TASK_IAM_ROLE=true trong ecs.config
  Tip: Fargate không cần ecs.config

[Câu] Terminate instance STOPPED — vẫn hiện cluster
  Yêu cầu đề: Terminate EC2 container instance nhưng vẫn thấy trong cluster
  Đáp án: STOPPED → agent mất heartbeat, registration không deregister sạch
  Tip: Drain → deregister → terminate; hoặc deregister-container-instance --force

[Câu] Centralized logging (Fargate)
  Đáp án: awslogs trong task definition
  Tip: Metrics ≠ logs; không cài agent trên Fargate; không cần sidecar cho logs

[Câu] CodeBuild push ECR fail (authorization)
  Đáp án: IAM service role CodeBuild thiếu quyền ECR
  Tip: Authorization = IAM, không phải Security Group

[Câu] Pull image từ ECR — local test (chọn 2)
  Yêu cầu đề: Developer pull Docker image từ repo ECR `demo` để test local
  Yêu cầu đề: Chọn 2 lệnh bắt buộc — login registry rồi pull full URI
  Đáp án (2): $(aws ecr get-login --no-include-email) + docker pull {account}.dkr.ecr.{region}.amazonaws.com/demo:latest
  Tip: CLI v2 thực tế: aws ecr get-login-password | docker login … — exam vẫn hay đưa get-login
  Tip: Không docker login -u $AWS_ACCESS_KEY_ID -p $AWS_SECRET_ACCESS_KEY (ECR dùng token tạm)
  Tip: docker build = build local; aws docker push = không tồn tại
  Lỗi hay nhầm: Login bằng access key/secret trực tiếp

6. ELASTIC BEANSTALK
--------------------------------------------------------------------------------

[Câu] EB keywords — Application, Environment, Application Version
  Yêu cầu đề: Phân biệt khái niệm cơ bản EB trước khi chọn deploy/migrate
  Yêu cầu đề: Application = container logic; Environment = bản chạy thật có URL/health
  Đáp án: Application chứa nhiều Environment + Application Version (zip code trên S3)
  Tip: Saved Configuration = snapshot cấu hình YAML — khác Application Version (source code)
  Tip: Save config path S3: resources/templates/{application-name}/

[Câu] Environment tier — Web server vs Worker
  Yêu cầu đề: Chọn loại environment khi create EB — web HTTP vs xử lý background
  Yêu cầu đề: Worker nhận job từ SQS; Web server có URL public + ALB (Traffic splitting, Blue/Green web)
  Đáp án: Web server (Standard) = HTTP/HTTPS cho user | Worker (SQS/HTTP) = poll SQS, không phục vụ web trực tiếp
  Tip: Console: Environment tier → Web server environment / Worker environment
  Tip: Saved Config YAML: EnvironmentTier Name WebServer hoặc Worker, Type Standard hoặc SQS/HTTP
  Tip: Traffic splitting, ALB canary — chỉ Web server tier

[Câu] 5 chiến lược deploy — tổng quan
  Yêu cầu đề: Zero downtime, rollback, canary, giữ capacity — chọn deployment policy
  Đáp án: All at once | Rolling | Rolling + additional batch | Immutable | Traffic splitting
  Tip: Blue/Green = 2 environment swap URL — khác 5 policy trong 1 env
  Tip: Traffic splitting chỉ ALB

[Câu] All at once
  Yêu cầu đề: Dev/test, deploy nhanh, chấp nhận downtime
  Đáp án: Cập nhật tất cả instance cùng lúc
  Tip: Production ít dùng; rollback = deploy lại version cũ

[Câu] Rolling
  Yêu cầu đề: Zero downtime, chấp nhận giảm capacity tạm thời
  Đáp án: Deploy từng batch; capacity giảm trong lúc update
  Tip: Kết hợp Deployment Groups chia % instance

[Câu] Rolling with additional batch
  Yêu cầu đề: Zero downtime + giữ full capacity; chấp nhận thêm instance tạm
  Đáp án: Launch batch mới trước → chuyển traffic → terminate batch cũ
  Tip: Khác Rolling thường — Rolling giảm capacity

[Câu] Immutable
  Yêu cầu đề: Rollback nhanh terminate fleet mới; tránh nửa cũ nửa mới trên instance
  Đáp án: Fleet EC2 mới hoàn toàn → swap → xóa fleet cũ
  Tip: An toàn trong 5 policy; rollback = terminate new fleet

[Câu] Traffic splitting — Canary
  Yêu cầu đề: 10% traffic test version mới; environment có ALB
  Đáp án: Traffic splitting — % traffic canary rồi tăng dần
  Tip: Chỉ ALB; Classic LB không hỗ trợ

[Câu] Cross-account migration
  Đáp án: Saved Config → download → sửa → upload S3 account B
  Tip: Không share qua IAM role

[Câu] ELB unhealthy, ASG không thay
  Đáp án: HealthCheckType: ELB trong .ebextensions
  Tip: EC2 check ≠ app health

[Câu] Min downtime + rollback nhanh
  Yêu cầu đề: Production swap DNS instant rollback; 2 environment song song
  Đáp án: Blue/Green + swap Route 53 CNAME / EB swap URL
  Tip: Rollback = swap DNS lại; khác Immutable terminate fleet

[Câu] Chọn chiến lược deploy — decision tree
  Yêu cầu đề: Scenario → chọn đúng 1 policy
  Đáp án: Dev→All at once; giảm capacity OK→Rolling; giữ capacity→Rolling+additional/Immutable
  Tip: Canary ALB→Traffic splitting; instant DNS rollback→Blue/Green

[Câu] Deploy lần lượt nhóm EC2
  Yêu cầu đề: Chia nhóm deploy tuần tự; không nhầm Hooks
  Đáp án: Deployment Groups + Rolling policy batch size
  Tip: Hooks = lifecycle script trên instance

[Câu] RDS tạo trong EB, destroy env
  Đáp án: Snapshot trước khi destroy
  Tip: Production: RDS standalone

[Câu] Chạm giới hạn 1000 app versions
  Đáp án: Application Version Lifecycle Policy
  Tip: MaxCount hoặc MaxAge

[Câu] 3 containers trên 1 EC2
  Đáp án: Docker multi-container (Dockerrun.aws.json v2)
  Tip: Single-container = 1 container/EC2

[Câu] EB HTTPS endpoint — in-flight encryption client ↔ web server
  Yêu cầu đề: Muốn HTTPS thay HTTP trên Beanstalk; client encrypt tới web server (terminate tại ALB)
  Yêu cầu đề: Chọn cách setup HTTPS đúng trên Beanstalk
  Đáp án: Tạo config file trong `.ebextensions` để cấu hình Load Balancer (listener 443 + ACM cert ARN)
  Tip: `aws:elbv2:listener:443` + `SSLCertificateArns` — cert ACM cùng region
  Lỗi hay nhầm: Health check — không bật HTTPS; CFN template riêng — EB quản lý LB; mở port 80 SG — HTTP không phải HTTPS

[Câu] ASP.NET IIS — nhiều version: dev, test, load test
  Yêu cầu đề: Nhiều version/môi trường chạy song song trên EB; dev/test vs load test gần production
  Yêu cầu đề: Chọn cách cấu hình deployment/process đúng pattern EB
  Đáp án: 1 Application — env dev (single instance) + env load-test (cấu hình gần production)
  Tip: Mỗi environment = URL + capacity + config riêng; deploy application version lên từng env
  Lỗi hay nhầm: ALB route hostname + ebextensions (overkill); chỉ 1 dev env; 1 env + Ansible

9. DYNAMODB
--------------------------------------------------------------------------------

[Câu] Latency cao, user global (chọn 2) — chat marathon
  Yêu cầu đề: User global latency cao; chọn 2 giải pháp
  Đáp án: Global Tables + Eventually consistent reads
  Tip: Connection pooling = SAI (DDB không như RDS); DAX ≠ giải pháp global latency

[Câu] Cross-account access — chat marathon
  Yêu cầu đề: Lambda A → DynamoDB B; không resource policy
  Đáp án: AssumeRole; trust policy trên role Account B

[Câu] API cơ bản — PutItem / GetItem / UpdateItem / DeleteItem
  Yêu cầu đề: Thao tác 1 item; ghi vs patch attribute
  Đáp án: PutItem overwrite; GetItem đọc 1; UpdateItem patch; DeleteItem xóa 1
  Tip: ConditionExpression optimistic lock trên Put/Update/Delete

[Câu] BatchGetItem & BatchWriteItem
  Yêu cầu đề: Nhiều item ít request; UnprocessedKeys/Items retry
  Đáp án: BatchGet ≤100/16MB; BatchWrite ≤25/16MB
  Tip: BatchWrite partial success; ACID → TransactWriteItems

[Câu] Query vs Scan
  Yêu cầu đề: Tìm theo PK hiệu quả vs đọc cả table
  Đáp án: Query bắt buộc PK; Scan đọc hết — đắt RCU
  Tip: Scan+Filter tốn RCU theo data scanned

[Câu] projection / filter / condition expression — chat practice
  Yêu cầu đề: Scan chỉ trả vài cột name email
  Đáp án: ProjectionExpression=attributes; FilterExpression=lọc items; ConditionExpression=điều kiện ghi
  Tip: filter-expression KHÔNG chọn cột — chỉ lọc item

[Câu] DAX — DynamoDB Accelerator
  Yêu cầu đề: Microsecond read; giảm RCU read-heavy
  Đáp án: DAX cluster in-memory; read-through + write-through
  Tip: DAX ≠ Global Tables; DAX ≠ chỉ write; API-compatible với DDB

[Câu] GSI vs LSI
  Yêu cầu đề: Query attribute không phải PK
  Đáp án: GSI PK/SK mới capacity riêng; LSI cùng PK khác SK lúc create table

[Câu] DynamoDB Streams + Lambda

  Đáp án: Enable Stream → Lambda event source mapping
  Tip: Stream per-table ordered; khác Kinesis

[Câu] TTL — tự xóa item hết hạn
  Yêu cầu đề: Session/log auto-delete; attribute Number epoch
  Đáp án: Enable TTL attribute; xóa nền không tốn WCU

[Câu] TransactWrite / TransactGet — ACID
  Yêu cầu đề: All-or-nothing nhiều item
  Đáp án: TransactWriteItems/TransactGetItems max 25, 4MB; write 2× WCU
  Tip: BatchWrite có thể partial success

[Câu] DynamoDB API — bảng tra cứu
  Đáp án: Put/Get/Update/Delete; BatchGet 100 BatchWrite 25; Query/Scan; Transact; PartiQL
  Tip: Pagination LastEvaluatedKey

[Câu] RCU / WCU — công thức cơ bản
  Yêu cầu đề: Cho kích thước item KB + read/write → hỏi cần bao nhiêu RCU/WCU
  Đáp án: WCU = ceil(KB/1); Strong RCU = ceil(KB/4); Eventual RCU = Strong ÷ 2
  Tip: 1 WCU = 1 KB write/s; 1 RCU strong = 4 KB read/s; 1 RCU eventual = 8 KB read/s
  Tip: Luôn ceil; transactional write = 2 × WCU

[Câu] Tính WCU — item lớn hơn 1 KB
  Yêu cầu đề: Item 2.5 KB, 6 KB ghi 1 lần/s → mấy WCU? BatchWrite cộng từng item
  Đáp án: WCU = ceil(kích thước KB); TransactWriteItems = 2 × WCU
  Tip: 2.5 KB → 3 WCU (không làm tròn xuống 2)

[Câu] Tính RCU — Strong vs Eventually consistent
  Yêu cầu đề: Cùng item so sánh RCU strong vs eventual; Scan/Query tính theo data đọc
  Đáp án: Strong RCU = ceil(KB/4); Eventual = Strong ÷ 2
  Tip: 10 KB strong = 3 RCU; eventual = 1.5 RCU
  Tip: Scan đọc cả table → RCU theo size scanned, không theo số row trả về

[Câu] Provisioned capacity — throttling
  Yêu cầu đề: ProvisionedThroughputExceededException; traffic spike vượt RCU/WCU
  Đáp án: Tăng RCU/WCU, Auto Scaling, hoặc chuyển On-Demand
  Tip: On-Demand tự scale, trả theo request; Provisioned rẻ hơn nếu traffic ổn định

[Câu] GSI — capacity riêng
  Yêu cầu đề: Query nhiều trên GSI; write vào table có GSI
  Đáp án: GSI có RCU/WCU riêng; PutItem tốn WCU table + WCU GSI (projection)
  Tip: Thiếu GSI capacity → throttle riêng, không kéo theo base table

[Câu] KCL trên EC2 — max instance sau split shard (6 → 10)

  Đáp án: 10 EC2 (bằng số shard hiện tại)

[Câu] Firehose — delivery S3 / Redshift / OpenSearch

[Câu] Firehose — Lambda transform & buffering
